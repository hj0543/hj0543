## 📑 Development Philosophy
멘토링을 통해 정립한 저의 개발 원칙입니다. 모든 프로젝트에 아래 과정을 준수합니다.

1. **Issue-Driven Development**: 모든 기능 구현과 버그 수정은 Issue 발행부터 시작합니다. (WBS 기반 관리)
2. **Commit with 'Why'**: 무엇을 했는지가 아닌, '왜' 이 코드가 필요한지를 커밋 메시지에 담습니다.
3. **Strict Workflow**: Git-Flow 전략을 지향하며, PR을 통한 코드 리뷰와 개선 과정을 기록합니다.
4. **User-Centric Solution**: 기술적 도전이 실제 사용자에게 어떤 가치를 주는지 고민합니다.
